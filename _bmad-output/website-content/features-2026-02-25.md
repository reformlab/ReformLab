## Open Data Pipeline

### Connect to European data sources without the pain

Public data from INSEE, Eurostat, and EU-SILC contains everything you need for policy impact analysis. The problem is getting it into a usable state: downloading massive files, cleaning inconsistent formats, joining datasets that don't share identifiers, interpolating missing values, generating synthetic populations.

ReformLab handles this entire pipeline.

**What you get:**

- Pre-built connectors for French and European open data sources
- Automatic data harmonization into a consistent analytical schema
- Synthetic population generation from public marginals — no restricted microdata required
- Full traceability: every data transformation step is logged and inspectable

**What you skip:**

- Downloading and cleaning raw CSV/Parquet files manually
- Writing custom jointure scripts for every new analysis
- Building synthetic populations from scratch using IPF or custom code
- Documenting your data pipeline after the fact

## Policy Scenario Templates

### Define policies in minutes, not weeks

Instead of encoding policy logic from scratch every time, start from proven templates and customize.

**Available templates:**

- **Carbon tax** — Define tax rate per tonne CO2, energy categories, exemptions, and redistribution mechanism (flat dividend, progressive rebate, tax credit)
- **Energy subsidies** — Renovation subsidies, equipment subsidies, energy efficiency programs with eligibility rules and benefit schedules
- **Feebates** — Combined fee-and-rebate structures for vehicles, appliances, or housing
- **Targeted rebates** — Income-tested transfers with customizable thresholds and phase-out schedules
- **Social transfers** — General-purpose transfer templates compatible with OpenFisca tax-benefit calculations

**How templates work:**

1. Pick a template that matches your policy type
2. Adjust parameters: tax rates, thresholds, eligibility rules, redistribution formulas
3. Define reform scenarios as diffs against a baseline — change one parameter, keep everything else
4. Run immediately or save for comparison

Templates are YAML files. You can edit them in the GUI or in a text editor. Every template is fully documented with parameter descriptions and default values.

## Dynamic Orchestrator

### Multi-year projections with real state tracking

Most microsimulation tools give you a single-year snapshot. ReformLab runs projections over 10+ years with explicit year-to-year state transitions.

**What the orchestrator does:**

- Executes a pluggable step pipeline between yearly computations
- Carries forward household state year to year (income changes, household composition, asset ownership)
- Applies vintage transitions: heating systems age, vehicles are replaced, buildings are renovated — tracked by cohort and year of acquisition
- Computes cumulative indicators across the full projection horizon

**Why this matters:**

Environmental policies play out over years, not months. A carbon tax phased in over five years has different distributional effects than one implemented overnight. Renovation subsidies only make sense when you track which buildings have been renovated and when. Vehicle emission standards only bite when the fleet turns over.

The orchestrator makes these dynamics visible and computable.

**Technical detail:** The orchestrator wraps OpenFisca's yearly computations. OpenFisca handles tax-benefit logic for each year; the orchestrator handles everything between years — state carry-forward, vintage updates, and (Phase 2) behavioral response adjustments.

## Scenario Comparison

### Test policy variants side by side

Define a baseline and one or more reform scenarios. Run them all. See the differences instantly.

**Comparison capabilities:**

- **Side-by-side indicators** — Distributional charts, welfare tables, fiscal cost estimates for each scenario on the same screen
- **Diff view** — Highlight exactly what changed between scenarios: which parameters differ, which populations are affected differently
- **Parameter sweeps** — Select a parameter (e.g., carbon tax rate), define a range, and see how indicators shift across the range
- **Winners and losers tables** — For each scenario, see which income deciles, regions, or household types gain or lose — and by how much

**Use case:** Sophie's director asks: "What happens if we set the carbon tax at 44, 67, or 100 euros per tonne, with three different redistribution options?" Sophie defines 9 scenario combinations in YAML, runs them all, and has a comparison matrix ready.

## Indicators and Analysis

### From raw numbers to decision-ready outputs

Every simulation run produces a structured set of indicators designed for policy briefings and academic publications.

**Standard indicator set:**

- **Distributional impact** — Effects by income decile, quintile, or custom grouping
- **Welfare indicators** — Equivalent variation, compensating variation, consumer surplus changes
- **Fiscal indicators** — Revenue, cost, net fiscal impact of reforms
- **Policy performance** — Targeting efficiency, coverage rates, benefit adequacy
- **Geographic breakdowns** — Regional and departmental disaggregation (where data permits)

**Output formats:**

- Visual charts in the GUI and notebooks (matplotlib, built-in chart components)
- CSV/Parquet export for downstream analysis
- Executive summary tables ready for briefing documents

**Every indicator links back to its source.** Click a number in a chart and trace it: which policy formula produced it, which population slice fed it, which synthetic data assumptions built that slice, which source tables were used.

## Run Manifests and Reproducibility

### Every run is traceable. Every result is reproducible.

When you run a simulation, ReformLab automatically generates an immutable manifest documenting everything about that run.

**What a manifest contains:**

- Engine version and all dependency versions
- Input data file hashes (so you know the exact data used)
- All policy parameters and scenario configuration
- All synthetic population assumptions and generation parameters
- Timestamp and execution environment details

**What this enables:**

- **Reproduce any result** — Re-run a manifest on any machine and get identical outputs
- **Audit any assumption** — Every parameter, every default, every data source is documented
- **Share with confidence** — Send a manifest and notebook to a colleague; they get the same results
- **Defend your work** — When someone questions a number, show them exactly where it came from

**No manual documentation required.** The manifest is generated automatically as a side effect of running the simulation. You don't write it — the system does.

## No-Code GUI

### Analysis without engineering

The GUI is the primary interface — not an afterthought bolted onto a command-line tool.

**What you can do in the GUI:**

- Select data sources and synthetic population configurations
- Choose or customize policy scenario templates
- Launch simulation runs with one click
- Compare scenarios side by side with visual indicators
- Inspect run manifests and trace results back to assumptions
- Export results as CSV, Parquet, or chart images

**What you can't do in the GUI (use the Python API instead):**

- Write custom policy formulas beyond template parameters
- Build entirely new entity models
- Integrate with external tools or pipelines programmatically

The GUI and the Python API share the same objects. Anything configured in the GUI can be exported as a notebook. Anything built in code can be loaded into the GUI.

## Python API and Jupyter Integration

### Full programmability for researchers

For users who prefer code, ReformLab exposes a complete Python API.

```python
from reformlab import Scenario, Population, run

# Load pre-built synthetic population
pop = Population.from_template("france_households_2024")

# Define a carbon tax reform
scenario = Scenario.from_template("carbon_tax",
    tax_rate=100,  # EUR/tCO2
    redistribution="progressive_rebate"
)

# Run and get results
results = run(scenario, pop)
results.plot_distributional()  # Income decile chart
results.manifest.save("my_run_manifest.json")
```

**API features:**

- Load populations and scenarios from templates or custom configurations
- Run simulations and access results as pandas/Polars DataFrames
- Generate all standard indicators programmatically
- Export run manifests for reproducibility
- Full Jupyter notebook integration with rich display representations

**Designed for research workflows:** Results display cleanly in notebook cells. Charts render inline. Manifests are serializable. Everything you need for a replication package.

## Open Source and Extensible

### Bring your own models, data, and ideas

ReformLab is open-source and designed for extensibility.

**What you can customize:**

- **Data sources** — Connect your own datasets alongside built-in open data pipelines
- **Policy models** — Add custom policy formulas or extend existing templates
- **Indicators** — Define custom indicators beyond the standard set
- **Population generation** — Use alternative synthetic population methods or restricted microdata (kept local, never transmitted)

**Built on open standards:**

- OpenFisca for tax-benefit computation
- CSV/Parquet at all data boundaries
- YAML for policy configuration
- Python scientific stack (NumPy, pandas/Polars, matplotlib)
- No cloud dependency — runs fully offline on a standard laptop
