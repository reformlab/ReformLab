## 2) What Went Well

- Strong architecture layering emerged early: `loaders` -> `methods` -> `pipeline` -> `validation` -> notebook demo.
- Review-and-fix loop was effective: multiple critical review findings were closed before Epic completion.
- Test-first hardening worked:
  - Loader behavior now covers offline mode, stale cache fallback, schema enforcement, encoding edge cases, and network error wrapping.
  - Merge methods and validation include deterministic and edge-case checks.
- Documentation quality improved during implementation:
  - Method assumptions are explicit and human-readable.
  - Story anti-pattern docs captured lessons quickly enough to affect later stories in the same epic.

## 3) Challenges and Friction

- Data fidelity pitfalls were real:
  - Story 11.2 initially mixed declared/disposable income semantics and required corrective fixes.
- Error-boundary mistakes repeated:
  - Several review findings across loaders involved exception handling that could mask root causes or trigger wrong stale-cache behavior.
- Test intent drift appeared multiple times:
  - "Lying tests" and weak assertions were recurring issues in review artifacts.
- Artifact consistency drift:
  - `epics.md` still marks Epic 11 (and 12) as backlog while `sprint-status.yaml` shows completed stories.
  - Story status vocabulary is inconsistent (`done`, `complete`, `dev-complete`), which complicates automation and reporting.

## 4) Cross-Story Lessons

1. Domain-correct fixture data matters as much as schema-correct data.
   - Correct types and column names are insufficient when semantic mapping is wrong.
2. Fallback behavior must be strictly scoped.
   - Network fallback and stale-cache reuse should never hide parser/validation defects.
3. Assertions must validate behavior, not just code paths.
   - Tests need to prove the requirement, not only execute related functions.
4. Contracts between planning docs and implementation artifacts need automated checks.
   - Manual drift between `epics.md` and sprint tracking creates avoidable confusion.
