## Findings

1. **CRITICAL - Path traversal / arbitrary file execution surface**  
   The resolver trusts `descriptor.json["data_file"]` and joins it directly with `folder_path`; `../` escapes are not blocked and file extension is not constrained.  
   Evidence: [population_resolver.py:249](/Users/lucas/Workspace/reformlab/src/reformlab/server/population_resolver.py:249), [population_resolver.py:250](/Users/lucas/Workspace/reformlab/src/reformlab/server/population_resolver.py:250).  
   Fix: resolve + `relative_to(folder_path.resolve())` check, and enforce suffix in `{".csv",".parquet"}`.

2. **HIGH - Determinism violation in resolver precedence**  
   `_DATA_EXTENSIONS` is a `frozenset`; iteration order is not a stable contract across runs. If both `id.csv` and `id.parquet` exist, selected dataset can drift.  
   Evidence: [population_resolver.py:29](/Users/lucas/Workspace/reformlab/src/reformlab/server/population_resolver.py:29), [population_resolver.py:185](/Users/lucas/Workspace/reformlab/src/reformlab/server/population_resolver.py:185), [population_resolver.py:207](/Users/lucas/Workspace/reformlab/src/reformlab/server/population_resolver.py:207).  
   Fix: use ordered tuple, e.g. `(".parquet", ".csv")` or explicit policy.

3. **HIGH - Task claim mismatch: route does not use dependency injection for resolver**  
   Story notes claim integration via dependency, but `run_simulation` does not declare `resolver: PopulationResolver = Depends(get_population_resolver)` and instead calls global getter inside function.  
   Evidence: [runs.py:43](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/runs.py:43), [runs.py:75](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/runs.py:75).  
   Impact: tighter coupling, harder testing, weaker architecture compliance.

4. **HIGH - Failure metadata regression on 404 population resolution**  
   Population resolution happens before the `try/finally` metadata-save block. Missing population returns 404 and exits without persisted failure metadata, contradicting route doc behavior (“auto-saves on success and failure”).  
   Evidence: [runs.py:79](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/runs.py:79), [runs.py:98](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/runs.py:98).  
   Fix: move resolution into guarded execution path and persist failed metadata for this class of execution-blocking errors.

5. **IMPORTANT - Type integrity hole for `population_source`**  
   `ResultMetadata.population_source` is `str | None` and `_dict_to_metadata` accepts any string. Downstream response models expect a strict literal set. Corrupt metadata can produce runtime validation errors.  
   Evidence: [result_store.py:80](/Users/lucas/Workspace/reformlab/src/reformlab/server/result_store.py:80), [result_store.py:443](/Users/lucas/Workspace/reformlab/src/reformlab/server/result_store.py:443).  
   Fix: validate/coerce to `Literal["bundled","uploaded","generated"] | None` at load boundary.

6. **IMPORTANT - AC1/AC2/AC3 are under-verified (lying integration tests)**  
   Tests only assert HTTP 200 and `population_source`; they never assert that the resolved file path/data was actually consumed by execution.  
   Evidence: [test_runs.py:418](/Users/lucas/Workspace/reformlab/tests/server/test_runs.py:418), [test_runs.py:472](/Users/lucas/Workspace/reformlab/tests/server/test_runs.py:472), [test_runs.py:531](/Users/lucas/Workspace/reformlab/tests/server/test_runs.py:531).
