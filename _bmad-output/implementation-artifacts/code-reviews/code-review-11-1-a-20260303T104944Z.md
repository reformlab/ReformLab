## 🏛️ Architectural Sins

- **Boundary breach (cache key scope):** Cached data are keyed by URL+params, but stale fallback ignores params and returns any cached file in the dataset folder, breaking data-contract isolation when multiple parameterized variants exist.  
  - 📍 `src/reformlab/population/loaders/cache.py:70-106`  
  - 💡 Fix: When falling back, filter candidates whose metadata `params` exactly match the requested `config.params` before reuse.

- **Stale selection policy:** Stale fallback returns the alphabetically first file, not the newest, so offline runs may serve arbitrarily old data.  
  - 📍 `src/reformlab/population/loaders/cache.py:88-104`  
  - 💡 Fix: Load metadata for all stale candidates and pick the latest `downloaded_at`; surface staleness age in `CacheStatus`.

---

## 🐍 Pythonic Crimes & Readability

- **Missing mandated future import:** Project rule requires every Python file to start with `from __future__ import annotations`; empty package initializers omit it.  
  - 📍 `tests/population/__init__.py`; `tests/population/loaders/__init__.py`  
  - 💡 Fix: Add the import at top of each `__init__.py` (even if otherwise empty).

---

## ⚡ Performance & Scalability

- **Double I/O on cache miss:** `CachedLoader.download()` unconditionally reads any stale cache (via `SourceCache.get`) before attempting `_fetch`, causing a full disk read even when fresh download will occur. For large datasets this halves throughput and increases memory pressure.  
  - 📍 `src/reformlab/population/loaders/base.py:110-133`  
  - 💡 Fix: Add a `peek_status()` that avoids loading table data; only load stale table when you actually need to return it.

---

## 🐛 Correctness & Safety

- **Param-agnostic stale reuse (data corruption risk):** Stale fallback ignores `config.params`, so requesting a different year or filter can silently return mismatched data from an older cached variant.  
  - 📍 `src/reformlab/population/loaders/cache.py:70-106`  
  - 🔄 Reproduction: Cache dataset with params A, then request same provider/dataset_id with params B and no fresh cache; stale path returns A.

## 🔧 Maintainability Issues

- **Staleness ambiguity:** Because stale selection is arbitrary, reproducibility and governance logs can be misleading—runs may consume non-deterministic cache content depending on filesystem order.  
  - 📍 `src/reformlab/population/loaders/cache.py:88-104`  
  - 💥 Explosion radius: Offline runs and network-fail paths become non-reproducible across machines.
