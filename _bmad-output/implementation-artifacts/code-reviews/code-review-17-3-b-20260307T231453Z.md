## 🔧 Maintainability Issues

- **💣 Tech Debt:** `ResultCache` lacks a `delete` method. The `ResultCache` is intended to be an in-memory LRU cache for `SimulationResult` objects. While `DELETE /api/results/{run_id}` successfully removes metadata from the persistent store, it cannot remove the corresponding `SimulationResult` from the `ResultCache`. This means that if a result is deleted, it might still be retrieved from the in-memory cache until it's naturally evicted, leading to a discrepancy between the persistent state and the in-memory state. This will require a future cleanup or a `ResultCache.delete()` implementation to ensure consistency.
  - 📍 `src/reformlab/server/dependencies.py`
  - 💥 Explosion radius: High - potential for serving stale or deleted data, requiring manual cache invalidation or waiting for LRU eviction.
