## Findings (Ordered by Severity)

1. **CRITICAL: Load failure still links scenario to portfolio (AC-5, Task 3.5/6.2 violation).**  
`loadPortfolioIntoComposition()` catches and suppresses errors, but `handleLoad()` always runs `updateScenarioField("portfolioName", name)` and success toast afterward.  
Impact: scenario can be marked complete with a portfolio that never loaded; user gets false success.

2. **CRITICAL: Parameter edits are dropped on save (AC-1/AC-4 integrity break).**  
In save payload construction, policies are sent with `extra_params: {}`; composition parameter edits (`e.parameters`) are not serialized.  
Impact: round-trip save/load does not preserve what user configured inline.

3. **HIGH: Validation errors are treated as zero conflicts.**  
`runValidation()` catch block sets `conflicts` to empty without surfacing a validation-error state.  
Impact: validity indicator can flip green on backend failure; conflict safety is bypassed.

4. **HIGH: Debounced validation race allows invalid save.**  
Validation runs 500ms later; save checks current `conflicts` only. Immediate save after composition change can slip through before conflict detection executes.  
Impact: `resolutionStrategy="error"` does not reliably block conflicting portfolios.

5. **MEDIUM: React state anti-pattern in `handleRemove`.**  
Calling `setSelectedTemplateIds()` from inside `setComposition()` updater introduces side effects in updater function.  
Impact: brittle under StrictMode/concurrent rendering; hard-to-reason state ordering.

6. **MEDIUM: Nav completion logic accepts empty string as “complete.”**  
Policies completion checks `!== null && !== undefined`; `""` passes.  
Impact: false completion checkmark with invalid portfolio reference.

7. **MEDIUM: Missing adversarial tests for critical failure paths.**  
Story tests focus on happy path; missing assertions that failed load does **not** update scenario link and failed/pending validation blocks save.  
Impact: regressions for AC-3/AC-5 are not guarded.

## Suggested Fixes
1. Make `loadPortfolioIntoComposition` return success/failure (or rethrow), and only update scenario state + success toast on success.
2. Serialize `composition.parameters` into request payload (likely `extra_params` or canonical backend field) and validate round-trip.
3. Add `validationError` state; when validation fails, force invalid status for `composition.length >= 2` with `resolutionStrategy="error"`.
4. In `handleSave`, run synchronous validation guard (or await immediate validate) before create.
5. Refactor composition/template selection state to `useReducer` or separate pure updater logic.
6. Tighten nav check to non-empty string: `typeof portfolioName === "string" && portfolioName.trim().length > 0`.
7. Add tests for failed `getPortfolio`, validation failure, and save-before-debounce race.
