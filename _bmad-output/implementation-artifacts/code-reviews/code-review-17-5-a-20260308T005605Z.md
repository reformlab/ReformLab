## Correctness & Safety Findings

1. **Critical: No-data/no-income UX paths are non-functional in real runtime.**  
   `extractErrorDetail()` expects `err.body.detail`, but `apiFetch` throws `ApiError` with top-level `what/why/fix`. Result: AC-1 info state and AC-3 no-income disable path do not trigger correctly.

2. **AC-4 gap: transition chart is not clickable by year.**  
   Only table rows call `onYearClick`; the chart has no click handler.

3. **AC-4 gap: eligibility summary is never returned by backend.**  
   Domain summaries are hardcoded with `eligibility=None`, despite decision records carrying eligibility data.

4. **Unhandled input bug: invalid decile string returns 500.**  
   `group_value="abc"` crashes during `int()` cast.

5. **Validation gap: unsupported `group_by` silently returns unfiltered data.**  
   Typos/unknown grouping values are not rejected.

6. **Fail-loud contract breach: missing chosen column gets silently zeroed.**  
   Corrupted/incomplete panel data is converted into plausible-but-false output.

7. **Performance issue: avoid Python materialization of Arrow columns at scale.**  
   `to_pylist()` on large probability/year columns and nested filtering loops will degrade latency/memory.

---

## Test Quality Findings

8. **Frontend test suite is asserting a fake error contract.**  
   Tests pass with `{ body: { detail: ... } }`, which does not match production `ApiError`.

9. **Backend decile tests are too weak to catch ranking/partition defects.**  
   “`< 100` and `> 0`” is insufficient for correctness; no invalid-decile/input tests exist.

---

## Suggested Fixes

1. Update screen error handling to use `err instanceof ApiError` and read `err.what/err.why/err.fix`.
2. Add chart-level year click interaction (`AreaChart`/active payload click) to satisfy AC-4.
3. Propagate and aggregate domain eligibility into `DecisionSummaryResponse.domains[*].eligibility`.
4. Validate request model: `group_by` as `Literal["decile"] | None`, `group_value` constrained to `1..10`; return 422 on invalid.
5. Fail loudly when metadata declares a domain but `{domain}_chosen` is absent.
6. Strengthen tests:
   - frontend: reject with real `new ApiError(...)` objects;
   - backend: exact expected decile memberships and invalid-input 422 paths.
