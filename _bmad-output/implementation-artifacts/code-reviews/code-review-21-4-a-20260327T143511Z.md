## Findings (Ordered by Severity)

1. **Critical: Trust-boundary inversion accepted by API**
- The endpoint accepts both `(open-official, synthetic-public)` and `(synthetic-public, open-official)` pairs.
- That means a caller can pass `observed_id` as synthetic and still get HTTP 200.
- UI then hardcodes observed badge as non-synthetic and synthetic badge as synthetic, creating contradictory labeling.
- Evidence: [populations.py:1269](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:1269), [PopulationComparisonView.tsx:140](/Users/lucas/Workspace/reformlab/frontend/src/components/population/PopulationComparisonView.tsx:140), [PopulationComparisonView.tsx:155](/Users/lucas/Workspace/reformlab/frontend/src/components/population/PopulationComparisonView.tsx:155)
- Fix: enforce `observed_id -> open-official` and `synthetic_id -> synthetic-public` strictly; reject swapped input with 422.

2. **Critical: Malformed descriptor is silently downgraded instead of failing loudly**
- `_get_descriptor()` catches any descriptor parse error and silently falls back to synthetic-default descriptor.
- AC4 explicitly requires 422 for malformed `descriptor.json`.
- This is also a trust-governance bypass (bad metadata becomes accepted metadata).
- Evidence: [populations.py:1208](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:1208), [populations.py:1212](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:1212), [story:64](/Users/lucas/Workspace/reformlab/_bmad-output/implementation-artifacts/21-4-add-public-synthetic-asset-ingestion-and-observed-versus-synthetic-comparison-flows.md:64)
- Fix: convert parse failure directly into structured 422 (`what/why/fix`) and stop fallback.

3. **High: Story 21.4 comparison flow is not actually reachable in the app**
- `PopulationComparisonView` exists but is never routed from population stage.
- `comparison` subview in `App.tsx` points to run-comparison dashboard, not observed-vs-synthetic comparison.
- `AppContext` has no `comparisonState` for selected observed/synthetic IDs as story tasks require.
- Evidence: [App.tsx:159](/Users/lucas/Workspace/reformlab/frontend/src/App.tsx:159), [PopulationStageScreen.tsx:199](/Users/lucas/Workspace/reformlab/frontend/src/components/screens/PopulationStageScreen.tsx:199), [AppContext.tsx:140](/Users/lucas/Workspace/reformlab/frontend/src/contexts/AppContext.tsx:140)
- Fix: add `comparisonState`, route `#population/comparison`, and modal/button flow from population library.

4. **High: AC9 test requirements for compare flow are not implemented**
- Missing `tests/data/test_comparison.py`.
- No backend tests for `GET /api/populations/compare` status matrix (404/400/422/malformed descriptor).
- No frontend tests for `TrustStatusBadge`, `SyntheticBadge`, and `PopulationComparisonView` error states.
- Evidence: [story:361](/Users/lucas/Workspace/reformlab/_bmad-output/implementation-artifacts/21-4-add-public-synthetic-asset-ingestion-and-observed-versus-synthetic-comparison-flows.md:361), [test_populations_api.py:1](/Users/lucas/Workspace/reformlab/tests/server/test_populations_api.py:1)
- Fix: add explicit Story 21.4 test modules and route/component assertions.

5. **High: Resolver inconsistency breaks folder-based populations outside compare endpoint**
- `_scan_populations_with_origin()` supports folder populations (`data/populations/<id>/data.parquet`).
- `_find_population_file()` does not; it only checks flat files (`<id>.csv/.parquet`).
- Result: items can appear in list but fail in preview/profile/crosstab/delete.
- Evidence: [populations.py:102](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:102), [populations.py:313](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:313)
- Fix: unify all endpoints on a single resolver that handles both folder-based and file-based populations.

6. **High: Type contract drift hidden by `any` casts in trust-critical UI path**
- `SyntheticBadgeProps.origin` is typed as legacy origin (`built-in|generated|uploaded`) but logic compares canonical origins.
- Population comparison view uses `as any` for trust labels to force compilation.
- This defeats TypeScript guarantees exactly where trust labels matter.
- Evidence: [SyntheticBadge.tsx:15](/Users/lucas/Workspace/reformlab/frontend/src/components/population/SyntheticBadge.tsx:15), [SyntheticBadge.tsx:22](/Users/lucas/Workspace/reformlab/frontend/src/components/population/SyntheticBadge.tsx:22), [PopulationComparisonView.tsx:138](/Users/lucas/Workspace/reformlab/frontend/src/components/population/PopulationComparisonView.tsx:138)
- Fix: use canonical literal types in `PopulationComparisonResponse` and badge props; remove all `as any`.

7. **Medium: Trust tooltip requirement is missing**
- Story trust-boundary rules require tooltip visibility for trust status.
- `TrustStatusBadge` renders plain badges without tooltip metadata.
- Evidence: [TrustStatusBadge.tsx:17](/Users/lucas/Workspace/reformlab/frontend/src/components/population/TrustStatusBadge.tsx:17), [story:289](/Users/lucas/Workspace/reformlab/_bmad-output/implementation-artifacts/21-4-add-public-synthetic-asset-ingestion-and-observed-versus-synthetic-comparison-flows.md:289)

- That scan loads tables to derive column counts, even though compare needs only two selected populations.
- Evidence: [populations.py:1216](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:1216), [populations.py:347](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:347), [populations.py:427](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/populations.py:427)
