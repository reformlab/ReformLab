**File:** `src/reformlab/templates/energy_poverty_aid/compute.py`
**Issue:** If the `energy_expenditure` column exists but contains data that cannot be cast to `float64`, the `try...except` block silently converts all values to 0.0, making all households ineligible without a clear error or warning log. This masks potential data quality issues.
