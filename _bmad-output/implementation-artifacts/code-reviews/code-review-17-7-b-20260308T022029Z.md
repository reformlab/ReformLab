- **[IMPORTANT] Inefficient File Export:** The `_file_response` helper in `exports.py` reads the entire export file into memory before streaming. This defeats the purpose of `StreamingResponse` for large files and can lead to significant memory usage on the server.
  - 📍 `src/reformlab/server/routes/exports.py:32`
  - 💡 Fix: Modify `_file_response` to stream the file content directly using `FileResponse` or a generator, avoiding loading the entire file into memory. This will require careful handling of the temporary directory lifecycle.

- **🐛 Bug:** Hardcoded Success Status in Disk Loading. The `SimulationResult` reconstructed by `ResultStore.load_from_disk` is always marked `success=True`. This is incorrect for runs that failed internally but still produced a `panel_output`, leading to misleading status when loaded from disk.
  - 📍 `src/reformlab/server/result_store.py:350`
  - 🔄 Reproduction: Simulate a scenario where `run_scenario` produces a `SimulationResult` with `success=False` but `panel_output` is not `None`. Store it, then restart the application and load it via `cache.get_or_load`. The loaded `SimulationResult.success` will incorrectly be `True`.
