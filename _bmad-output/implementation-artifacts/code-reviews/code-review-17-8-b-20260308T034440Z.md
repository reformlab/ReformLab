  - 📍 `frontend/src/App.tsx`
  - 💡 Fix: Extract navigation logic into a dedicated router or state machine. Break down into smaller, more focused components.
- **[9/10] Single Responsibility:** `frontend/src/components/screens/ComparisonDashboardScreen.tsx` (and other screen components) are excessively large and contain logic for multiple distinct sub-components (e.g., `RunSelector`, `FiscalTab`, `WelfareTab`, `DetailPanel` are defined within it).
  - 📍 `frontend/src/components/screens/ComparisonDashboardScreen.tsx`

  - 📍 `src/reformlab/server/models.py`, `frontend/src/api/types.ts`

This directly violates the project's critical rule: "All domain types are frozen — never add a mutable dataclass." This exposes the system to potential unintended mutations and non-deterministic behavior.
  - 📍 `src/reformlab/server/result_store.py`

## ⚡ Performance &amp; Scalability

- **Inefficient PyArrow Operations:** In `src/reformlab/server/routes/decisions.py`, the `_build_domain_summary` and `_filter_to_decile` functions chain multiple PyArrow table filters and computations within a loop over years. While PyArrow is highly optimized, for very large datasets, repeatedly filtering and recomputing derived columns might be less efficient than a single pass or pre-computing masks.
  - 📍 `src/reformlab/server/routes/decisions.py`
  - 💡 Fix: Investigate if a single, more complex filter expression or pre-computation of masks across years could optimize performance for extremely large panel outputs.

## 🐛 Correctness &amp; Safety

- **Resource Leak (Potential):** In `frontend/src/api/results.ts`, the `exportResultCsv` and `exportResultParquet` functions use `URL.createObjectURL`. While `URL.revokeObjectURL` is called in the happy path, it might not be guaranteed in all error paths (e.g., if `anchor.click()` throws an error or `response.blob()` fails asynchronously), potentially leading to minor memory leaks of Blob URLs.
  - 📍 `frontend/src/api/results.ts`
  - 🔄 Reproduction: Simulate `anchor.click()` or blob creation errors to confirm if `URL.revokeObjectURL` is consistently called.
- **Lying Tests (API Mocking):** The `frontend/src/__tests__/workflows/analyst-journey.test.tsx` extensively mocks *all* API modules used by `AppContext`. This means the tests primarily verify frontend component interaction with mocked API responses, not actual end-to-end integration with the backend. If the real API changes its contract or behavior, these tests will likely still pass, giving a false sense of security.
  - 📍 `frontend/src/__tests__/workflows/analyst-journey.test.tsx` (all tests within this file)
  - 🤥 Why it lies: Mocks entire API layer, testing mocks not the real integration path.
