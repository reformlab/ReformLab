## 🏛️ Architectural Sins

### Single Responsibility Principle - Violated

- **[5/10] Unclear parameter semantics:** `portfolioName` accepts `string | null` but empty string `""` has different semantics than `null`, leading to divergent behavior paths that callers may not expect.
  - 📍 `naming.ts:191, 200-203`
  - 💡 Fix: Use `string | null | undefined` with explicit handling: `if (portfolioName?.trim())` instead of `if (portfolioName)`

- **[4/10] Responsibility confusion:** The function handles three distinct concerns: 1) portfolio display names, 2) composition-based generation, 3) generic "Untitled" fallbacks - mixing these creates cognitive overhead for callers.
  - 📍 `naming.ts:190-216`
  - 💡 Fix: Split into separate functions: `useDisplayName()` for existing portfolios, `generateFromComposition()` for new scenarios
