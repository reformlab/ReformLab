## 🏛️ Architectural Sins

No pure SOLID violations, but two design boundary issues:

- **Missing update path in save hook:** `usePortfolioSaveDialog` has a documented design intent (Dev Notes: "If name exists and matches loaded portfolio: allow overwrite") but `handleSave` calls only `createPortfolio` — never `updatePortfolio`. The hook has access to `loadedPortfolioRef.current` and could branch on it. Currently the normal "re-save loaded policy set" workflow is completely broken.
  - 📍 `frontend/src/hooks/usePortfolioSaveDialog.ts:108-134`
  - 💡 Fix: Check `portfolioSaveName === loadedPortfolioRef.current`; if true, call `updatePortfolio` instead of `createPortfolio`.

- **Metadata persistence not symmetric with update:** `_save_portfolio_metadata` is called conditionally (`if metadata:`) on update but the old file is never deleted when the condition is false. This means removing all editable groups from an existing policy set leaves a stale `metadata.json` that will be re-hydrated on next load.
  - 📍 `src/reformlab/server/routes/portfolios.py:612-615`
  - 💡 Fix: Always write (or explicitly delete) `metadata.json` on `update_portfolio`, even when the new metadata is empty.

## 🐍 Pythonic Crimes & Readability

**Type Safety — frontend test mocks use wrong field names:**

The `Template` interface (`mock-data.ts:17-19`) defines `parameterCount: number` and `parameterGroups: string[]` (camelCase). The name-freeze test file constructs `Template[]` literals with `parameter_count` and `parameter_groups` (snake_case). These fields do not exist on the `Template` type.

- 📍 `frontend/src/hooks/__tests__/usePortfolioSaveDialog.nameFreeze.test.ts:46-47, 53-54, 64-65, 70-71`
- The tests pass at runtime because `generatePortfolioSuggestion` only reads `template.name` and `t.id`, but `npm run typecheck` (mypy equivalent: TypeScript strict) **should fail** on these object literals. The story claims all quality gates pass — this is contradictory and needs verification.

**Non-deterministic ID generation violating project rules:**

```typescript
id: `group-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`,
```
- 📍 `frontend/src/components/screens/PoliciesStageScreen.tsx:313`
- `Math.random()` is non-deterministic, contradicting the project rule "All runs must be deterministic and reproducible." Even as UI state, this makes snapshot tests impossible and complicates reproducibility.
- `String.substr()` is deprecated (MDN: "may be removed in the future"). Use `String.substring()`.
- 💡 Fix: Use the `instanceCounterRef` pattern already established in this file, e.g. `group-${instanceCounterRef.current++}`.
