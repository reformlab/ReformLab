## 🏛️ Architectural Sins

- **[8/10] Dependency Inversion Violation - Multiple Registry Singletons:** Two routes (`templates.py` and `portfolios.py`) create their own `ScenarioRegistry` instances instead of using the centralized dependency injection in `dependencies.py`.
  - 📍 `src/reformlab/server/routes/templates.py:72-78`
  - 📍 `src/reformlab/server/routes/portfolios.py:247-250`
  - 💡 Fix: Use `from reformlab.server.dependencies import get_registry` and remove local `_get_registry()` functions. This ensures all routes share the same registry instance.

- **[7/10] Open/Closed Principle Violation - Hardcoded Policy Classification:** The runtime availability classifier uses hardcoded sets (`LIVE_READY_TYPES`, `HIDDEN_PACK_TYPES`) that require manual updates when new policy types are added.
  - 📍 `src/reformlab/server/routes/templates.py:29-30`
  - 💡 Fix: Move availability classification to a policy metadata system where each template declares its own availability status, making the classifier data-driven rather than hardcoded.

✅ No abstraction-level issues found beyond the SOLID violations.

## 🐍 Pythonic Crimes & Readability

- **Type Safety - Global Singleton Without Locking:** Module-level singleton `_registry` is initialized without thread synchronization. FastAPI is async, so concurrent requests could trigger race conditions during initialization.
  - 📍 `src/reformlab/server/routes/templates.py:72-78`
  - 💡 Fix: Use `threading.Lock` or `asyncio.Lock` to synchronize initialization, or use the already-thread-safe dependency injection in `dependencies.py`.

- **Style - Broad Exception Catching Swallows Errors silently:** In `_load_builtin_packs()`, catching `(TemplateError, ScenarioError, OSError)` and continuing silently means corrupted YAML files disappear from the catalog without any indication.
  - 📍 `src/reformlab/server/routes/templates.py:161`
  - 💡 Fix: Log the specific exception details and/or track skipped templates separately so admins can identify and fix corrupted files.

- **Documentation Debt - Story References in Error Messages:** The availability reason `"Domain translation pending - see Story 24.2"` creates documentation that will become stale after Story 24.2 is complete.
  - 📍 `src/reformlab/server/routes/templates.py:60`
  - 💡 Fix: Use a more timeless message like `"Domain translation pending - requires variable mapping"` or reference a canonical documentation URL.
