## Findings (Ordered by Severity)

1. **CRITICAL: Task completion false claim (Task 2.2 not implemented).**  
   Task says add `store: ResultStore = Depends(get_result_store)` to `run_simulation()`. Actual code still calls `get_result_store()` inside `finally`, bypassing DI.  
   Source: [runs.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/runs.py)

2. **HIGH: Export route error-contract/security regression on invalid `run_id`.**  
   `_lookup_run()` catches only `ResultNotFound`; path-validation failures (`ResultStoreError`) bubble to 500. Prior story patterns require 400 structured response for invalid IDs.  
   Source: [exports.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/exports.py)

3. **HIGH: Corrupt persisted panel metadata is silently accepted.**  
   `load_panel()` swallows JSON decode failure for `reformlab_panel_metadata` and returns empty metadata. This violates fail-loud data contract and can break decision/indicator behavior after restart without obvious error.  
   Source: [result_store.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/result_store.py)

4. **HIGH: Reconstructed result correctness bug (`success=True` hardcoded).**  
   `load_from_disk()` always returns `SimulationResult(success=True, ...)` instead of reflecting persisted metadata status. A stale/partial artifact can be represented as successful.  
   Source: [result_store.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/result_store.py)

5. **HIGH: Atomic write is incomplete on failure paths.**  
   `save_panel()` / `save_manifest()` use tmp+replace but do not cleanup tmp files on write/replace exceptions. This creates orphan temp artifacts and operational drift over time.  
   Source: [result_store.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/result_store.py)

6. **MEDIUM: LRU eviction bug in `ResultCache.store()`.**  
   On full cache, method evicts first, then writes key. If key already exists, this still evicts another run unnecessarily (silent retention loss).  
   Source: [dependencies.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/dependencies.py)

7. **MEDIUM: Test suite misses the actual AC-critical integration path.**  
   Tests validate `ResultStore` primitives and cache loading but do not assert that `POST /api/runs` persists `panel.parquet` + `manifest.json` nor that panel/manifest save failures never alter API success behavior.  
   Source: [test_cache_disk_loading.py](/Users/lucas/Workspace/reformlab/tests/server/test_cache_disk_loading.py), [test_result_store_persistence.py](/Users/lucas/Workspace/reformlab/tests/server/test_result_store_persistence.py)

8. **MINOR: Story traceability/documentation mismatch.**  
   Story “File List” is tightly scoped, but provided diff indicates large unrelated modifications; this weakens auditability for Story 17.7 acceptance evidence.  
   Source: `git-diff` virtual context

## Suggested Fixes

1. Inject `ResultStore` into `run_simulation()` via `Depends(get_result_store)` and remove direct `get_result_store()` call.
2. In `routes/exports.py`, catch `ResultStoreError` and return structured 400 for invalid `run_id`.
3. In `load_panel()`, treat corrupt metadata JSON as `ResultStoreError` (do not silently coerce).
4. In `load_from_disk()`, set `success` from persisted metadata status (or explicitly derive with guard logic).
5. Add `try/finally` cleanup for `.panel.parquet.tmp` and `.manifest.json.tmp` on write failures.
6. Fix LRU overwrite logic: if key exists, update/move-to-end without eviction.
7. Add integration tests around `/api/runs` for:
   - successful panel+manifest persistence,
   - save failure non-propagation behavior,
   - restart-like retrieval using disk artifacts only.
