## 🐛 Correctness &amp; Safety

- **🐛 Bug:** Backend `portfolios.py` `_build_policy_config` function converts `exemptions`, `thresholds`, and `covered_categories` from lists to tuples with `tuple(req.exemptions)` (and similar for others). However, the Pydantic model for `PortfolioPolicyRequest` in `models.py` defines these as `list[str]` and `list[float]`. While `tuple` is immutable, passing a `list` to `tuple()` directly converts its elements. If `req.exemptions` contains elements that are *not* `str` (or `float` for thresholds), a runtime error will occur during the conversion from `list` to `tuple`, even if the Pydantic validation has already passed for the `list` type. This can happen if the Pydantic model allows `Any` or `Union` that resolves to a type that doesn't convert to string/float well.
  - 📍 `src/reformlab/server/routes/portfolios.py:228-230`
  - 🔄 Reproduction: Provide a `PortfolioPolicyRequest` where `exemptions` (or `thresholds`, `covered_categories`) contains a non-string (or non-float) element, and observe a runtime `TypeError` during tuple conversion.
