For large files (up to 500MB), this can consume significant memory, potentially impacting performance and scalability, especially under concurrent load. `StreamingResponse` typically works best by streaming data incrementally from a generator or file-like object without loading the entire content into RAM.
  - 📍 `src/reformlab/server/routes/exports.py`:24
  - 💡 Fix: Refactor `_file_response` to stream the file directly from disk using `FileResponse` or a generator with `StreamingResponse` to reduce memory footprint.

However, `apiFetch` and `apiFetchBlob` in `frontend/src/api/client.ts` are designed to throw an `ApiError` instance only when the API response contains a structured error (either top-level or nested under `detail`).
