## 🏛️ Architectural Sins

- **[10/10] Critical Return Type Bug — `handleMoveParameter`:**
  The outer `prev.map((entry, i) => {...})` callback reaches the `// Add to target group` block and executes `return newGroups.map(...)`. `newGroups.map(...)` evaluates to `EditableParameterGroup[]` — an array of group objects, not a `CompositionEntry`. React state then becomes `(CompositionEntry | EditableParameterGroup[])[]`. The next render hits `entry.templateId`, `entry.name`, etc. on the array object, producing `undefined` for every field and breaking the card entirely.
  - 📍 `frontend/src/components/screens/PoliciesStageScreen.tsx:304-311`
  - 💡 Fix: wrap the second map in the entry: `return { ...entry, editableParameterGroups: newGroups.map(...) };`

- **[9/10] Persistence Layer Never Writes or Reads Story 25.3/25.4 Fields:**
  `_portfolio_to_detail` constructs each `PortfolioPolicyItem` with exactly four fields: `name`, `policy_type`, `rate_schedule`, `parameters`. The `editable_parameter_groups`, `category_id`, and `parameter_groups` fields — all declared optional in both the Pydantic model and the `PortfolioPolicyItem` — are silently dropped on every save, and therefore always `null`/`[]` on load. AC-8 ("group names, order, and parameter memberships are restored correctly") is provably impossible to satisfy.
  - 📍 `src/reformlab/server/routes/portfolios.py:289-294`
  - 💡 Fix: extend `_portfolio_to_detail` to pass `category_id`, `parameter_groups`, and `editable_parameter_groups` through to `PortfolioPolicyItem`; extend the persistence domain object or sidecar JSON to carry these UI-layer fields.

## 🐛 Correctness & Safety

- **🐛 Bug — `handleMoveParameter` corrupts composition state:** Moving a parameter via the Select dropdown calls `handleMoveParameter`, which replaces `composition[policyIndex]` with `EditableParameterGroup[]`. All subsequent renders of that card access properties on an array (`entry.name`, `entry.templateId`, `entry.parameters`) and receive `undefined`. The card silently breaks. TypeScript should have caught the return type mismatch.
  - 📍 `frontend/src/components/screens/PoliciesStageScreen.tsx:290-312`
  - 🔄 Reproduction: Add two groups to a policy, add a parameter to group 0, enter edit mode, use the move dropdown to move the parameter to group 1 → the entire policy card immediately loses its name, badges, and controls.
