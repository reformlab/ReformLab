## 🏛️ Architectural Sins

- **[7/10] Single Responsibility Principle Violation:** The `makeDefaultAppState` helper in TopBar.test.tsx creates a 58-property mock object. This violates SRP by making tests responsible for understanding and maintaining the entire AppContext shape. When AppContext changes, every test breaks.
  - 📍 `frontend/src/components/layout/__tests__/TopBar.test.tsx:58-118`
  - 💡 Fix: Create a test factory function in a shared test utilities file that app can import. Tests should only override what they need.

- **[6/10] Open/Closed Principle Violation:** The external URLs are hardcoded strings in the component. Adding a third link requires modifying the component itself rather than extending it.
  - 📍 `frontend/src/components/layout/TopBar.tsx:76-92`
  - 💡 Fix: Create a `EXTERNAL_LINKS` configuration constant array and map over it. This allows adding links without touching the component.

- **[5/10] Abstraction Leakage:** The TopBar reaches into AppContext to grab 4 specific properties (`activeStage`, `activeScenario`, `saveCurrentScenario`, `apiConnected`). If AppContext changes its internal structure, TopBar breaks.
  - 📍 `frontend/src/components/layout/TopBar.tsx:25`
  - 💡 Fix: Create a targeted `useTopBarData` hook that AppContext exports explicitly. This creates a stable contract.

- **[4/10] Separation of Concerns Issue:** AC-3 claims scenario controls are in a "center-left container with visual separation" but the implementation uses a single flex container with `gap-x-4` that also contains the brand block, separator, AND stage label. The separation isn't scoped to just scenario controls.
  - 📍 `frontend/src/components/layout/TopBar.tsx:34-70`
  - 💡 Fix: Create an explicit `<div className="flex items-center gap-x-4">` that wraps ONLY the brand block and scenario controls, then put the separator and stage label outside this wrapper.

  - 📍 `frontend/src/components/layout/__tests__/TopBar.test.tsx:58-118`
