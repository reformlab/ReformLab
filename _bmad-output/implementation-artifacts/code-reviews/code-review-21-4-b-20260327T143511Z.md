- **[9/10] FastAPI Route Ordering Violation:** The `GET /compare` endpoint is defined AFTER the `DELETE /{population_id}` endpoint. FastAPI matches routes in definition order, so `/compare` will be captured by the `/{population_id}` parameter route first, making the comparison endpoint unreachable.
  - 📍 `src/reformlab/server/routes/populations.py:1157`
  - Lines 565 (`/{population_id}/preview`), 810 (`/{population_id}/profile`), 873 (`/{population_id}/crosstab`), 1097 (`DELETE /{population_id}`), then 1157 (`GET /compare`)
  - 💡 Fix: Move `@router.get("/compare")` BEFORE all `/{population_id}` routes (immediately after `@router.get("")` at line 539)

- **[7/10] Single Responsibility Principle (SRP) Violation:** `_scan_populations_with_origin()` is a 250+ line function that handles folder scanning, metadata reading, descriptor validation, canonical evidence mapping, and origin detection all in one place.
  - 📍 `src/reformlab/server/routes/populations.py:552-850`
  - 💡 Fix: Extract separate functions for `_read_folder_based_population()`, `_read_file_based_population()`, `_read_uploaded_population()`, `_get_descriptor_from_folder()`

  - 📍 `frontend/src/components/population/PopulationComparisonView.tsx:138,141,151,154`

  - 📍 `frontend/src/components/population/SyntheticBadge.tsx:15,22`
