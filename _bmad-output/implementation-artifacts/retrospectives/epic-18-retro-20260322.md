## Challenges and Growth Areas

### 1. 100% Pre-Synthesis Reject Rate (6th Consecutive Epic)
Every story received a REJECT or CHANGES REQUESTED verdict before synthesis correction. This pattern has persisted through Epics 13, 14, 15, 16, 17, and 18.

**Root cause:** The pre-submission checklist identified as the solution in Epic 17's retro was never implemented. Without it, stories consistently reach code review with mechanical gaps (accessibility attributes, edge case handling, test gaps for new async flows) that are reliably caught and fixed in synthesis.

**Impact:** Each story incurs a mandatory extra synthesis pass. Review rounds that should pass are failing. The code ships correct, but the process is inefficient and creates a false signal that quality is not improving.

**Notable instances this epic:**
- 18.3: scope creep (custom template modal added without tests) → one unresolved action item carried forward
- 18.4: stale detail race condition (run ID change not guarded in async fetch)
- 18.7: concepts panel state not reset on navigation (useEffect with deps missing)

### 2. Backend Regressions Carrying for 2 Epics
The two backend regressions identified in Epic 17's retro are now 2 epics old:
- **`RunRequest.parameters` / `policy` field mismatch** — simulation runner cannot pass policy params; the primary functional path is broken at the API boundary
- **`except Exception` broad catches in scenarios.py/templates.py** — silent 404s mask real errors

Epic 18 was a frontend-only epic, making deferral reasonable in isolation. But these issues have now accumulated through two delivery cycles and will block any backend integration work in Phase 3.

**Root cause:** No explicit "debt sprint" or mandatory fix window was defined after Epic 17.

### 3. Sprint Status Desync (Recurring)

### 4. Scope Creep Without Test Coverage (18.3)

## Key Insights

1. **The review cycle is not a quality problem — it's a process problem.** Code review consistently catches real issues (accessibility, race conditions, edge cases, state bugs), but these issues are also consistently fixable in one synthesis pass. The mechanism works; the pre-synthesis gate is missing.

2. **Frontend-only epics create backend debt accumulation.** When an epic is scoped purely to UI, backend regressions survive indefinitely. A mixed or alternating scope strategy — or a dedicated debt window after each epic — would prevent multi-epic accumulation.

3. **The 105 new tests add more value than the number suggests.** Many are covering previously-untested async paths, component state transitions, and accessibility attributes. They represent a quality floor that didn't exist before this epic.

4. **Component extraction creates compounding value.** The `components/comparison/` extraction in 18.5 directly enabled the clean delivery of contextual help in 18.7 (no right panel surgery needed). Epic 17's monolithic App.tsx would have made 18.7 much harder.

5. **Chart polish is cheap when centralized.** `chart-theme.ts` is 60 lines and affects 5 components. The CSS custom property pattern for chart colors enables trivial future theming. The upfront cost of standardization pays forward.

6. **Design decisions deferred in Epic 17 created tech debt in Epic 18.** Raw `<input>` elements, duplicate WorkbenchStepper implementations, and inline error displays were all "defer for now" decisions from Epic 17 that became targeted cleanup in 18.3/18.6. Tracking these at story-close would have created a cleaner handoff.
