### 1. ExogenousAsset Caching for Performance

**Value:** Avoid repeated sorting in get_value()

**Suggestion:**
The story correctly notes that `ExogenousAsset.get_value()` sorts years on every call. For scenarios with many years, this is inefficient. Suggest caching sorted years in the asset:

```python
@dataclass(frozen=True)
class ExogenousAsset:
    _sorted_years: tuple[int, ...] = field(init=False)
    
    def __post_init__(self):
        object.__setattr__(self, "_sorted_years", tuple(sorted(self.values.keys())))
```

### 2. Immutable _series Field Clarification

**Value:** Clearer intent for frozen dataclass

**Suggestion:**
The story uses `_series` (single underscore) for private field. In frozen dataclasses, all fields are technically immutable via reassignment. Consider documenting that single underscore indicates "internal use only" convention, not true privacy (Python doesn't have private fields).
