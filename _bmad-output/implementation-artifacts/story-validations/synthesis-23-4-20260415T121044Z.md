## Issues Dismissed

- **Silent downgrade via REFORMLAB_RUNTIME_MODE** | **Raised by**: Validator B | **Dismissal Reason**: Env var override is explicit operator control with startup logging, not a silent runtime downgrade. Requested "live" mode always uses live adapter; env var only affects requests without explicit runtime_mode. The story explicitly logs the adapter choice at startup.

- **API adapter inconsistency as bug** | **Raised by**: Validator A | **Dismissal Reason**: This is intentional design. Server uses MockAdapter for dev/test environments (matches existing patterns), while Python API uses SimpleCarbonTaxAdapter for demo/quickstart scenarios. Both are appropriate for their contexts. Added notes to clarify this distinction.

- **Circular dependency risk with constant placement** | **Raised by**: Validator A | **Dismissal Reason**: No actual circular import exists. The constant lives in result_normalizer.py and is imported by dependencies.py and api.py. This is a unidirectional dependency (dependencies → computation), not a circular one. The placement is logical since it derives from _DEFAULT_OUTPUT_MAPPING.

- **Excessively prescriptive implementation** | **Raised by**: Validator B | **Dismissal Reason**: Code examples are guidance, not rigid requirements. Developers can adapt patterns as needed. The story focuses on outcomes (correct adapter selection) while providing helpful examples. The factory function names are suggestions, not mandates.

- **metadata.source values undefined** | **Raised by**: Validator A | **Dismissal Reason**: Out of scope for this story. Tests can verify behavior through metadata.runtime_mode (which Story 23.3 sets) rather than source field. The adapter version() method can be used if source verification is needed.

- **Initialize_default_adapter_for_direct purpose unclear** | **Raised by**: Validator A | **Dismissal Reason**: The function name and context make it clear this is for direct scenario execution from notebooks/scripts (not via web API). Added clarifying context in Dev Notes but this was not a blocker.

- **Portfolio run underspecified** | **Raised by**: Validator A | **Dismissal Reason**: Portfolio runs pass through the same run_simulation() function, so the runtime_mode logic applies automatically. The test requirement is sufficient to ensure portfolio runs respect the mode.
