### 5. Missing Handling of Policy Type Origin Ambiguity

**Impact:** The classification logic based on `policy_type` string value won't reliably distinguish between built-in templates and user-created scenarios using the same type. A user-saved scenario with `carbon_tax` type would be classified as `live_ready` even though it's a custom scenario that may not have live translation.

**Source:** Story file - Task "Create availability classifier function"

**Problem:**
The proposed classifier logic:
```python
if policy_type in ["carbon_tax", "subsidy", "rebate", "feebate"]:
    return "live_ready"
elif policy_type in ["vehicle_malus", "energy_poverty_aid"]:
    return "live_unavailable" with reason
else:
    return "live_unavailable"
```

This doesn't account for template origin:
- Built-in template from `carbon_tax/` directory → `live_ready` ✓
- User-saved scenario using `carbon_tax` type → incorrectly marked `live_ready` ✗
- Built-in custom type `vehicle_malus` → correctly marked `live_unavailable` ✓

The `ScenarioRegistry` stores user-saved scenarios, and these use the same `PolicyType` enum values as built-in templates. Without tracking origin, classification will be incorrect.

**Recommended Fix:**
Add detailed classification algorithm in dev notes:
```markdown
**Availability Classification Algorithm:**

The classifier must distinguish between THREE template sources:

1. **Built-in templates** (YAML files from packs/ directories):
   - Loaded via `_load_builtin_packs()` → `_load_builtin_template(name)`
   - Identified by: loaded from `src/reformlab/templates/packs/*.yaml`
   - Classification:
     * `carbon_tax`, `subsidy`, `rebate`, `feebate` → `"live_ready"`
     * `vehicle_malus`, `energy_poverty_aid` → `"live_unavailable"`

2. **User-saved scenarios** (from ScenarioRegistry):
   - Loaded via `registry.get(name)` in `get_template()` endpoint
   - Identified by: loaded from `ScenarioRegistry`, not from pack directory
   - Classification: `"live_unavailable"` (user scenarios not yet live-executable)

3. **Custom type registrations** (in-memory, created via /api/templates/custom):
   - Loaded via `list_custom_registrations()` in `list_templates()`
   - Identified by: in `_CUSTOM_POLICY_TYPES` dict, not in registry
   - Classification: `"live_unavailable"`

Implementation note: `_template_to_list_item()` receives `template` object and `name` 
string. We need additional context (source/origin) to classify correctly. Options:
- Add parameter to `_template_to_list_item(name, template, source)`
- Pass source from caller (built-in, registry, custom)

def classify_availability(policy_type: str, source: str) -> tuple[str, str | None]:

**Suggested Addition:**
```markdown
**Availability Classifier Implementation:**

Create a module-level function in `src/reformlab/server/routes/templates.py`:

```python
from typing import Literal

RuntimeAvailability = Literal["live_ready", "live_unavailable"]
LIVE_READY_TYPES = {"carbon_tax", "subsidy", "rebate", "feebate"}
HIDDEN_PACK_TYPES = {"vehicle_malus", "energy_poverty_aid"}

def classify_runtime_availability(
    policy_type: str, 
    is_custom_type: bool,
    is_registry_scenario: bool
) -> tuple[RuntimeAvailability, str | None]:
    """Determine runtime availability based on policy type and template origin.
    
    Args:
        policy_type: The template's policy type string value
        is_custom_type: True if using CustomPolicyType (not PolicyType enum)
        is_registry_scenario: True if loaded from ScenarioRegistry
    
    Returns:
        Tuple of (runtime_availability, availability_reason)
    """
    if is_custom_type or is_registry_scenario:
        # User-created content: not yet live-executable
        return "live_unavailable", None
    
    if policy_type in LIVE_READY_TYPES:
        return "live_ready", None
    
    if policy_type in HIDDEN_PACK_TYPES:
        return "live_unavailable", "Domain translation pending - see Story 24.2"
    
    # Fallback for unknown types
    return "live_unavailable", None
```

def _template_to_list_item(

    template: Any,
    source: Literal["builtin", "registry", "custom"] = "builtin"
) -> TemplateListItem:
    # ... existing parameter extraction code ...
    
    # Determine availability
    runtime_avail, reason = classify_runtime_availability(
        policy_type, 
        is_custom, 
        source == "registry"
    )
    
    return TemplateListItem(
        # ... existing fields ...
        runtime_availability=runtime_avail,
        availability_reason=reason
    )
