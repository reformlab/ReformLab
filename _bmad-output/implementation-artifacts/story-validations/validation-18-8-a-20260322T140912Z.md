### 1. AC-1: MultiRunChart `CHART_COLORS` Backward Compatibility

**Impact:** The story explicitly states in "Dev Notes" that `MultiRunChart.tsx` *re-exports* `CHART_COLORS` for backward compatibility. However, AC-1's instruction to ensure *all* chart colors reference CSS custom properties might be interpreted as removing the `CHART_COLORS` export if the agent strictly follows that one criterion. This could break external modules (`comparison-helpers.ts`, `RunSelector.tsx`) that import `CHART_COLORS` from `MultiRunChart.tsx`, leading to runtime errors.

**Source:** AC-1, Dev Notes (Backward Compatibility - Import Chain)

**Problem:** AC-1 states "No hardcoded hex color values remain in chart component source files." and implies all colors should *reference* CSS custom properties. While `chart-theme.ts` correctly defines `CHART_COLORS` using CSS variables, the task for `MultiRunChart.tsx` (Task 4.1) instructs to "re-export `CHART_COLORS` for backward compatibility," but AC-1 doesn't explicitly *allow* for this re-export. A strict interpretation of AC-1 could lead an agent to inadvertently remove the re-export if it prioritizes the "no hardcoded hex" rule over implied backward compatibility. This creates a conflict or an implicit expectation not fully covered by the ACs themselves.

**Recommended Fix:** Clarify AC-1 or add an explicit AC ensuring backward compatibility for `CHART_COLORS` and `DECISION_COLORS` imports from `MultiRunChart.tsx` and `TransitionChart.tsx` respectively, or update the AC to explicitly state that the re-export is part of the solution.
