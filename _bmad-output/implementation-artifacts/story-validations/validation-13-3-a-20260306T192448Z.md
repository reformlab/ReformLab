### Acceptance Criteria Issues

- **Missing Criteria:** AC2 defines the aid computation but does not explicitly mention the critical edge case handling for `income <= 0`, `energy_expenditure <= 0`, or a missing `energy_expenditure` column, which are detailed later in the "Edge Case Handling" table.
  - *Quote:* "Households with `income <= income_ceiling` AND `energy_expenditure / income >= energy_share_threshold` are eligible and receive aid."
  - *Recommendation:* Incorporate explicit statements about how `income <= 0`, `energy_expenditure <= 0`, and missing `energy_expenditure` columns affect eligibility and aid computation directly into AC2, or refer to the "Edge Case Handling" table from within the AC.
- **Ambiguous Criteria:** AC2 describes the `income_ratio` formula as `(income_ceiling - income) / income_ceiling`. It does not explicitly state how to handle the edge case where `income_ceiling` might be zero, which would lead to a division by zero error. While the default `income_ceiling` is 11000.0, this constraint isn't explicit for the formula.
  - *Quote:* "`income_ratio = (income_ceiling - income) / income_ceiling`"
  - *Recommendation:* Clarify the behavior if `income_ceiling` is 0 (e.g., raise an error, assume a default, or state it's a pre-validated positive value).
- **Missing Criteria:** AC2 could explicitly mention the specific behavior for `income = income_ceiling`, which results in `aid = 0`. This is a boundary condition in the formula.
  - *Quote:* "Households with `income <= income_ceiling` AND `energy_expenditure / income >= energy_share_threshold` are eligible and receive aid."
  - *Recommendation:* Add a clause to AC2 stating: "Households with `income = income_ceiling` will have an `income_ratio` of 0 and thus receive 0 aid (consistent with the eligibility condition)."
- **Missing Criteria:** AC2 mentions that null values are handled by `pc.fill_null()` in the Edge Case Handling table, but not in the AC itself. This is a critical implementation detail that ensures robustness.
  - *Quote:* "Ineligible households receive 0 aid." (From AC2). "Null values in income/energy_expenditure | Fill nulls with 0 via `pc.fill_null()`" (From Edge Case Handling Table)
  - *Recommendation:* Add a point to AC2: "Null values in `income` or `energy_expenditure` columns are treated as 0 for computation purposes, ensuring graceful handling via `pc.fill_null()`."

### Hidden Risks and Dependencies

- **Data Dependency:** The computation relies on the presence of an `energy_expenditure` column in the population table. Although the story gracefully handles a missing column (treating it as 0 expenditure), this implies a critical dependency on upstream data ingestion to provide this column for the template to be effective.
  - *Impact:* If the `energy_expenditure` column is consistently missing in actual data, the template will always yield zero aid, rendering it functionally useless despite technically "working".
  - *Mitigation:* Explicitly state the importance of the `energy_expenditure` column's availability in the input population data and consider if a warning or specific error is needed if it's absent, rather than silently treating it as zero. This could be a process-level mitigation rather than a story change.
