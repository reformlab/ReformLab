### 2. No scipy Optimization Failure Handling for Multi-Dimensional Case

**Impact:** CRITICAL - Production failures when optimization fails
**Source:** AC5, Task 5

**Problem:**
Multi-dimensional optimization is much more prone to convergence failures than scalar optimization. The story doesn't specify what to do when scipy.optimize.minimize fails:
- Return the best solution found?
- Raise CalibrationOptimizationError with diagnostic info?
- Retry with a different method?
- Fall back to initial values?

Without this specification, the system may crash or return invalid results in production.

**Recommended Fix:**
Add to AC5:
```
- Handles scipy optimization failures:
  - If result.success is False, raise CalibrationOptimizationError with:
    - Convergence status (result.message)
    - Final objective value
    - Number of iterations
    - Suggested remediation (e.g., "try different method", "relax bounds")
  - Log warning if gradient_norm > threshold (possible local minimum)
  - Provide CalibrationResult with partial results if result.success is False but a feasible solution exists
```
