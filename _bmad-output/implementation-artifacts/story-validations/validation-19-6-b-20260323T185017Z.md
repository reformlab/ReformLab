## ✨ Optimizations (Nice to Have)

### 1. `startTour()` Should Destroy Prior Instance Before Creating New One

**Value:** Defensive hygiene; prevents potential ghost instances in automated testing or edge cases.

**Suggestion:**
Add `driverRef.current?.destroy();` at the start of `startTour()` before calling `driver()`:

```tsx
const startTour = useCallback(() => {
  driverRef.current?.destroy();  // ← guard
  const driverInstance = driver({ ... });
  driverRef.current = driverInstance;
  driverInstance.drive();
}, []);
```

While the overlay makes the button unreachable during an active tour in normal use, this is best practice for ref-managed instances and prevents issues in automated testing.

### 2. Pre-specify `popover.side` for Left-Edge Nodes

**Value:** Prevents popovers for Population/Policy rendering off-screen at narrow viewports.

**Suggestion:**
Population (x=20, y=30) and Policy (x=20, y=150) are at the left edge of the SVG. driver.js auto-positions popovers based on available space, but at the left edge, it may render a popover further left, clipping against the viewport. Pre-specify `popover: { side: 'right' }` for steps 1 and 2:

```tsx
{
  element: '[aria-label="Population — click to learn more"]',
  popover: {
    title: 'Population',
    description: '...',
    side: 'right',  // ← push popover to the right of the node
  },
},
```

### 3. Recommend `data-tour-step` as Primary Selector (Not Fallback)

**Value:** Eliminates character-encoding brittleness in CSS attribute selectors.

**Suggestion:**
The story uses `[aria-label="Population — click to learn more"]` as the primary selector, with `data-tour-step` as a fallback. However, em-dash characters in attribute selectors can behave unexpectedly across copy-paste contexts (e.g., different Unicode code points when writing the selector). Since the story already documents modifying `DomainModelDiagram.tsx` as acceptable for this fallback, consider making `data-tour-step="population"` the primary selector to eliminate ambiguity. Add `data-tour-step` attributes to the 6 `<g>` nodes during implementation, avoiding the em-dash risk entirely.
