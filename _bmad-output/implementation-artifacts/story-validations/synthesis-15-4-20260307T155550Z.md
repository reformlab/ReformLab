- **Claimed Issue**: `TypeError` for type validation is inconsistent with subsystem exception policy | **Raised by**: Validator A | **Dismissal Reason**: Project context prohibits bare `Exception`/`ValueError` for *domain errors*. `TypeError` for wrong argument type is standard Python convention and acceptable for public API type guards. "Per domain" in AC-1 describes what field is recorded, not a batch interface. Callers wanting multi-domain capture call the function once per `CalibrationResult`. No API change needed.
- **Claimed Issue**: Deterministic entry ordering should be in AC, not just Dev Notes | **Raised by**: Validator A | **Dismissal Reason**: Ordering is an implementation detail already fully specified in Dev Notes (sort by key).
