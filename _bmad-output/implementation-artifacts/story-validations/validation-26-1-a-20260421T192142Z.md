### 2. Hash Migration Potential Infinite Loop

**Impact:** HIGH - The recommended hash migration implementation in AppContext.tsx could cause infinite recursion if the hashchange event fires while processing the migration.

**Source:** Story file "File: frontend/src/contexts/AppContext.tsx" section, lines showing migration code

**Problem:**
The story suggests this implementation:
```typescript
// Update hash if migration occurred
if (stage === "engine") {
  window.location.hash = sub ? `scenario/${sub}` : "scenario";
}
```

This triggers a new `hashchange` event. If the effect runs again before the hash is actually updated in the browser, or if there's any timing issue, the code could re-enter the migration logic infinitely.

**Recommended Fix:**
Use the one-time migration flag approach shown in the "Alternative approach" section of the story, and make it the PRIMARY recommendation:
```typescript
const engineMigratedRef = useRef(false);

// Inside onHashChange:
if (stage === "engine" && !engineMigratedRef.current) {
  engineMigratedRef.current = true;
  window.location.hash = sub ? `scenario/${sub}` : "scenario";
  return; // Prevent double-processing
}
```
