### 2. `data_available` Rule Can Be Wrong

**Impact:** UI can show run data as available when panel is actually unavailable (failed run cached).  
**Source:** [17-7 story](/Users/lucas/Workspace/reformlab/_bmad-output/implementation-artifacts/17-7-implement-persistent-result-storage-and-retrieval.md), [results.py](/Users/lucas/Workspace/reformlab/src/reformlab/server/routes/results.py)

**Problem:** Proposed rule uses `cache.get(meta.run_id) is not None`, which ignores `panel_output is None`.

**Recommended Fix:** Define exact formula: `has_panel_on_disk OR (cache_hit AND cache_hit.panel_output is not None)`.

**Impact:** Disk-read failures may produce 500s or inconsistent behavior across endpoints.
