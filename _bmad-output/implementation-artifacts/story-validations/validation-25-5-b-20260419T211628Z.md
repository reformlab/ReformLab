## 🚨 Critical Issues (Must Fix)

### 1. Contradictory Name-Freeze Semantics

**Impact:** Developers can implement opposite behaviors and both claim compliance; tests become unstable.  
**Source:** AC-3 vs Known Issues note (“per-save-dialog-session”).

**Problem:** Story simultaneously implies persistent manual override across later composition changes and session-scoped recalculation after dialog close.

**Recommended Fix:** Add one explicit rule:
- Option A: Manual name persists across dialog reopen until user presses “Reset suggestion”.
- Option B: Manual name only persists while dialog is open.
Then align tasks and tests to that one rule.

### 2. Undefined `version_id` Contract

**Impact:** Backend/front-end contract drift and false “done” status.  
**Source:** AC-1.

**Problem:** Story requires `version_id` but does not define where it comes from, its type, format, or endpoint payload changes.

**Recommended Fix:** Add API contract table with exact JSON response for save/load/clone including `version_id` semantics and compatibility guarantees.

### 3. Policy Set Reference Semantics Are Unsafe

**Impact:** Scenario results may change unexpectedly over time, undermining reproducibility.  
**Source:** AC-5 + Out of Scope “no versioning”.

**Problem:** Name-based pointer without explicit version/snapshot policy leaves runtime behavior ambiguous.

**Recommended Fix:** Define one of:
- Pin scenario to policy set version at run creation, or
- Always resolve latest by name (explicitly non-reproducible, with warning), or
- Store both name + resolved hash/version in run manifest.

### 4. Migration Requirement Is Under-Specified

**Impact:** Real risk of silent data loss or duplicate-state corruption.  
**Source:** AC-6 + migration tasks.

**Problem:** No explicit legacy key map, precedence rules, or rollback behavior.

**Recommended Fix:** Add migration specification with:
- legacy keys and new keys,
- transform rules,
- idempotency rule,
- fallback if parse fails,
- test vectors (before/after JSON).
