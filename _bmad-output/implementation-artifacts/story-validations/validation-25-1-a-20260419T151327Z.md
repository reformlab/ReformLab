## 🚨 Critical Issues (Must Fix)

### 1. `category_id` will be silently dropped — `mock-data.ts` and `useApi.ts` are missing from the file list

**Impact:** The entire frontend grouping/filtering feature (AC2, AC3) will not work. `category_id` will exist in the API response but will be stripped at the `mapTemplate()` boundary before reaching the component.

**Source:** `frontend/src/hooks/useApi.ts:132-145`, `frontend/src/data/mock-data.ts:13-24`

**Problem:**
The template data flows through a mapping layer:
```
GET /api/templates → TemplateListItem[] → mapTemplate() → Template[] → AppContext → PortfolioTemplateBrowser
```
`mapTemplate()` (line 132–145 of `useApi.ts`) explicitly maps only the fields present in the `Template` interface from `mock-data.ts`. Adding `category_id` to `TemplateListItem` in `types.ts` is necessary but **not sufficient** — the field is dropped in `mapTemplate()` unless `Template` is also updated AND `mapTemplate()` is updated to pass it through.

Neither `frontend/src/data/mock-data.ts` nor `frontend/src/hooks/useApi.ts` appears in the story's File List or Tasks. A developer following the story exactly would miss both files.

**Recommended Fix:**

Add to File List:
- `frontend/src/data/mock-data.ts` — add `category_id?: string` to `Template` interface
- `frontend/src/hooks/useApi.ts` — add `category_id: item.category_id,` to `mapTemplate()`

Add to Tasks under "Frontend API wrapper":
- [ ] Add `category_id?: string` to `Template` interface in `frontend/src/data/mock-data.ts`
- [ ] Update `mapTemplate()` in `frontend/src/hooks/useApi.ts` to pass `category_id` through

**Suggested Addition:**
