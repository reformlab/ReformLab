### 2. Population Short Name Specification is Contradictory

**Impact:** Developer cannot implement correctly - examples conflict with description
**Source:** Task 3, Dev Notes

**Problem:**
Dev Notes state:
- Remove "France " prefix if present
- Remove year suffix if present  
- Example: "France Synthetic 2024" → "Synthetic 2024" or just "FR Synthetic"

These examples don't follow the stated rules:
- "Synthetic 2024" keeps the year (contradicts "remove year suffix")
- "FR Synthetic" changes "France" to "FR" (not mentioned in rules)

**Recommended Fix:**
Provide a single, consistent algorithm with testable examples:
```typescript
function getPopulationShortName(population: Population): string {
  // Remove "France " prefix
  let result = population.name.replace(/^France\s+/i, '');
  // Add "FR " prefix if not present
  if (!result.startsWith('FR')) {
    result = `FR ${result}`;
  }
  return result;
}
// "France Synthetic 2024" → "FR Synthetic 2024"
// "France Household Panel 2023" → "FR Household Panel 2023"
```

### 4. Demo Scenario Manual Edit Tracking Creates Circular Logic

**Impact:** Demo scenario will never receive auto-updates despite spec saying it shouldn't be "manually edited"
**Source:** Task 4, Dev Notes constraint #7

**Problem:**
Dev Notes state: "The demo scenario (created in `createDemoScenario()`) should NOT be treated as 'manually edited' so it can get auto-updates if portfolio/population context changes."

But `createDemoScenario()` returns a scenario with `name: "Demo — Carbon Tax + Dividend"` which is a manually crafted string, not auto-generated. How does the system know this isn't a manual edit?

**Recommended Fix:**
Add a special flag to demo scenario OR initialize the manually edited set with demo scenario ID excluded:
```tsx
const DEMO_SCENARIO_ID = "demo-carbon-tax-dividend";
// In effect:
if (activeScenario.id === DEMO_SCENARIO_ID) {
  // Never treat demo name as manual, always auto-update
  return;
}
```

### 5. Auto-Update Effect Fires Excessively

**Impact:** Performance issue and potential infinite loop scenario
**Source:** Task 4, Implementation Plan

**Problem:**
The proposed effect triggers on `portfolioName` and `populationIds` changes. Every time user changes portfolio in PoliciesStageScreen, the activeScenario name will update. This happens on every keystroke in the save dialog (since portfolio changes update the scenario).

**Recommended Fix:**
Add a debounce or only update when user navigates away from Policies stage:
```tsx
// Option 1: Debounce
const updateDebounced = useMemo(() => debounce((name) => {
  if (!activeScenario || manuallyEditedScenarioNames.has(activeScenario.id)) return;
  updateScenarioField("name", name);
}, 300), [activeScenario, manuallyEditedScenarioNames]);

// Option 2: Only update on stage navigation
useEffect(() => {
  if (activeStage !== 'policies') {
    // Update when leaving policies stage
  }
}, [activeStage]);
```
