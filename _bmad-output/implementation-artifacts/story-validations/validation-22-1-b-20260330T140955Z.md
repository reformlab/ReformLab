### 4. External Link Security Attributes Buried in Parenthetical

**Impact:** MEDIUM - Security requirement may be missed
**Source:** AC-2 parenthetical

**Problem:**
AC-2 mentions `target="_blank"` and `rel="noopener noreferrer"` inside parentheses:
> "when clicked, then they open the configured external destinations (`target="_blank"`, `rel="noopener noreferrer"`) in a new tab"

This critical security requirement is not explicit in the acceptance criterion and could be missed. The `rel="noopener noreferrer"` is a security best practice to prevent tabnabbing attacks when using `target="_blank"`.

**Recommended Fix:**
Rewrite AC-2 to make security explicit:
> "Given the website and GitHub link icons in the top bar, when clicked, then they open external URLs in a new tab AND include security attributes (target='_blank', rel='noopener noreferrer') for safe navigation."
