## ⚡ Enhancement Opportunities (Should Add)

### 1. Clarify `show()` helper function source

**Benefit:** Provides unambiguous instructions for implementing the `show()` helper, preventing developer confusion and ensuring consistency across notebooks.
**Source:** Dev Notes -> Notebook Cell Structure Pattern, Task 1.2, Anti-Patterns.

**Current Gap:** The Dev Notes contain slightly contradictory instructions regarding the `show()` helper function. It says "Alternatively, use `from reformlab import show` if available in the public API" and also "Copy the exact `show()` helper from `06_portfolio_comparison.ipynb`". If `show()` is intended to be copied, it shouldn't suggest importing from `reformlab`. If it's intended to be part of the public API, then the copy instruction is redundant.

**Suggested Addition:** Explicitly state the definitive source for the `show()` helper:
*   If `reformlab.show` is guaranteed to exist and be functional, then remove the instruction to copy and explicitly use the import.
*   Otherwise, remove the alternative import suggestion and unequivocally state that the `show()` helper function should be copied verbatim from `06_portfolio_comparison.ipynb` into the notebook's setup cell.

### 2. Refine AC-5 Wording for Clarity

**Benefit:** Improves the precision of an Acceptance Criterion, ensuring a clearer understanding of what the notebook should demonstrate regarding reproducibility.
**Source:** Acceptance Criteria

**Current Gap:** AC-5 refers to "YAML + notebook + manifest" as the sufficient components for independent reproduction. This is slightly inaccurate because the "notebook" is the demonstration vehicle, not part of the shared artifact for reproduction by a co-author. The shared artifact is the replication *package*, which contains JSON config files (derived from YAML) and the manifest.

**Suggested Addition:** Rephrase AC-5 to accurately reflect the components of the sharable artifact for reproduction. For example:
"**AC-5: Marco's sharing workflow** — Given the notebook output, when inspected, then it narratively demonstrates the researcher sharing workflow: Marco exports a replication package, a simulated "co-author" imports and reproduces it in a clean context, confirming that the exported package (containing configuration and manifest) is sufficient for independent reproduction."
