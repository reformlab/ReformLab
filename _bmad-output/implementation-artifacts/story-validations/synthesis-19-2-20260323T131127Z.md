- **Merge AC1 and AC10 into a single AC** | **Raised by:** Validator A (LLM optimization) | **Dismissal Reason:** They test different things. AC1 = Mermaid integration works. AC10 = overall build health. Keeping them separate preserves targeted regression detection.

- **Add formal link target matrix as AC** | **Raised by:** Validator A | **Dismissal Reason:** Dev Notes already contain a complete card table with link targets. Adding a duplicated matrix in ACs would create maintenance overhead with no additional clarity.

- **Add regression AC for sidebar/home navigation unchanged** | **Raised by:** Validator A | **Dismissal Reason:** Sidebar navigation regression is covered by Task 4's "verify all sidebar pages load correctly" subtask. A dedicated AC for this would be disproportionate.
