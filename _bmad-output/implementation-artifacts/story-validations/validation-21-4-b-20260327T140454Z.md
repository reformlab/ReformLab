### 4. Comparison Endpoint Loads Full Tables Into Memory

**Impact:** Comparing large populations (>100k rows) will cause memory issues or timeouts.

**Source:** Task 4

**Problem:**
Task 4 says "Loads tables via `_load_population_table()`" which reads entire Parquet/CSV into memory:
```python
# populations.py:170-175
def _load_population_table(file_path: Path) -> pa.Table:
    if file_path.suffix.lower() == ".parquet":
        return pa.parquet.read_table(file_path)  # Full table
```

For 100k row populations, comparison loads TWO tables simultaneously. With more populations or larger sizes, this becomes problematic.

**Recommended Fix:**
- Add performance requirement: "Comparison API must complete within 10 seconds for populations up to 100k rows"
- Specify chunking strategy for large populations or memory limits
- Add PyArrow streaming/zero-copy options where possible
- Consider computing statistics column-by-column instead of loading full tables
