## 🚨 Critical Issues (Must Fix)

### 1. Observed-vs-synthetic distinction is currently impossible

**Impact:** Core user outcome cannot be delivered correctly.  
**Source:** Story AC4/AC8 + current origin mapping behavior.

**Problem:** Current population origin mapping classifies built-in/generated/uploaded as synthetic-public, so “observed” is not reliably represented in population comparisons.

**Recommended Fix:** Add explicit AC requiring at least one open-official observed population with canonical metadata, and require API validation that one side is observed and the other synthetic.

### 2. Ingestion contract is internally inconsistent

**Impact:** High risk of building incompatible loaders and broken population discovery.  
**Source:** AC2/Task2 vs `load_population_folder()` and population route scanning behavior.

**Problem:** Story mixes folder ingestion with `descriptor.json` while existing loader expects `source.json`, and current population listing does not ingest folder datasets in this way.

**Recommended Fix:** Define authoritative ingestion contract: metadata file names, fallback order, schema validation behavior, and route integration acceptance tests.

### 3. Trust-boundary enforcement is not an acceptance requirement

**Impact:** Synthetic and observed data can be compared without guardrails, violating Epic trust goals.  
**Source:** AC4/AC8/AC10.

**Problem:** Endpoint AC does not require rejection of invalid pairings (synthetic-synthetic, observed-observed, missing trust metadata).

**Recommended Fix:** Add mandatory API guard AC: 422 for invalid pairings or missing canonical trust metadata, with explicit what/why/fix error response.

### 4. Decision-doc mandatory rules are incompletely encoded

**Impact:** Governance noncompliance despite “passing” story ACs.  
**Source:** Story trust rules quote + synthetic decision doc §3.5.

**Problem:** Story quotes requirement for `intended-use` and explicit restricted-source failure, but ACs do not require either.

**Recommended Fix:** Add ACs requiring intended-use display in comparison surfaces and explicit failure behavior for deferred restricted sources.
