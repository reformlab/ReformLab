## 🚨 Critical Issues (Must Fix)

### 1. Non-comparable tolerance semantics in side-by-side fit

**Impact:** Analysts can be shown misleading training vs holdout “within tolerance” status when `rate_tolerance` differs from training run tolerance.  
**Source:** AC-4, Dev Notes `validate_holdout()` step 7

**Problem:**  
`training_fit` is derived from `calibration_result.rate_comparisons` (precomputed with training tolerance), while `holdout_fit` uses current `rate_tolerance`. Side-by-side `all_within_tolerance` can become apples-to-oranges.

**Recommended Fix:**  
Require one of:
1. Recompute training `within_tolerance` using the passed `rate_tolerance`, or  
2. Return both tolerances explicitly and label them separately, or  
3. Disallow custom `rate_tolerance` in holdout validation and enforce calibration tolerance reuse.
