### 2. JSON Schema Compatibility Is Unresolved

**Impact:** Custom YAML may fail schema validation even if runtime registration exists.  
**Source:** Project Structure Notes, AC4

**Problem:** Story flags schema enum conflict as “consider whether,” not as a required decision and acceptance outcome.

**Recommended Fix:** Add AC: either dynamic schema support for custom values or a documented/implemented validation pathway that safely permits custom types.

### 3. Global Registration Lifecycle Is Not Specified

**Impact:** Order-dependent runtime behavior, test flakiness, and nondeterministic execution.  
**Source:** Key Design Decision 2, Task 7.13

**Problem:** State reset is mentioned for tests, but production lifecycle/thread/process expectations are missing.

**Recommended Fix:** Add AC for lifecycle semantics (registration phase, idempotency, conflict behavior, determinism) and required diagnostics.

### 4. Backward Compatibility Is Under-Specified

**Impact:** Existing built-in flows can silently regress while custom type tests pass.  
**Source:** Task 1.2, Task 7, AC2/AC5

**Problem:** “continue to work identically” is stated, but no exhaustive compatibility matrix is required.

**Recommended Fix:** Add explicit non-regression ACs covering load/serialize/portfolio/orchestrator for all built-in policy types.
