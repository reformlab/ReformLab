## 🚨 Critical Issues (Must Fix)

### 1. Missing Resolver Contract in Descriptor Spec

**Impact:** Quick Test can appear in list but fail at run-time selection.  
**Source:** story tasks + resolver behavior.

**Problem:** Story asks for `descriptor.json` metadata only, but resolver for folder populations requires `data_file`.  
**Recommended Fix:** Add explicit descriptor requirement: `data_file` (and expected filename), plus AC test that `POST /api/runs` succeeds with quick-test as `population_id`.

### 2. Household Count Requirement Cannot Be Met As Written

**Impact:** Scenario may show `0 households`, violating AC #4/#5.  
**Source:** scanner implementation.

**Problem:** `_scan_populations_with_origin()` infers households from ID tokens; `quick-test-population` has no numeric token.  
**Recommended Fix:** Add task to derive households from row count for folder populations (or consume explicit descriptor field and validate it).

### 3. Stage 2 Card Actions Likely Regress for Folder Population

**Impact:** Preview/Explore/Profile/Crosstab may fail for quick-test.  
**Source:** populations route path resolution.

**Problem:** `_find_population_file()` supports only flat files, not folder-based `data.parquet`.  
**Recommended Fix:** Require folder-aware resolution in all population data endpoints or route them through shared resolver logic.

### 4. Descriptor Minimal Spec Risks Compare Endpoint Failures

**Impact:** `/api/populations/compare` may return 422 for quick-test descriptor.  
**Source:** compare endpoint uses `DataAssetDescriptor.from_json`.

**Problem:** Story’s descriptor fields are incomplete for full descriptor validation.  
**Recommended Fix:** Require full descriptor schema compatibility (at least `asset_id`, `data_class`, taxonomy fields, and consistent metadata).

### 5. Canonical vs Legacy Origin Is Confused in Requirements

**Impact:** Wrong assertions and potentially wrong implementation.  
**Source:** task line “origin is synthetic-public”.

**Problem:** `synthetic-public` belongs to `canonical_origin`, not legacy `origin`.  
**Recommended Fix:** Update all requirements/tests to validate the correct field pair: `origin="built-in"` + `canonical_origin="synthetic-public"`.
