### Hidden Risks and Dependencies

- **Dependency management risk:** `astro-mermaid` is a community package (joesaby/astro-mermaid, 1 maintainer). The project's Astro 5.7.10 pin is older than the versions Dependabot PRs have tested (5.9.1–5.15.9). No version is pinned in the story. If the installed version pulls in a transitive dep that conflicts with the existing `zod: 3.25.76` override, the build will break in the same way the sitemap issue did in 19.1.
  - *Impact:* Build failure; could block the story entirely.

- **Client-side JavaScript requirement:** `astro-mermaid` renders diagrams client-side. If a user has JS disabled or uses a reader mode, the diagram will not render — only a raw code block.
  - *Impact:* Minor UX degradation; acceptable for v1 but worth documenting.
  - *Mitigation:* The story's risk table addresses this partially; no additional mitigation needed beyond noting it.

## 🚨 Critical Issues (Must Fix)

### 1. No version pin for `astro-mermaid`

**Impact:** Potential build failure from transitive dependency conflict (zod, shiki, etc.). Story 19.1 spent significant time resolving a dependency conflict introduced by `@astrojs/sitemap` pulling in zod v4.

**Source:** Task 1 / Dev Notes — `npm install astro-mermaid` with no version specified

**Problem:**
The story instructs `npm install astro-mermaid` without specifying a version. Story 19.1's Completion Notes document exactly this failure mode: an unversioned install of `@astrojs/sitemap` pulled in zod v4, causing a dual-instance conflict that required an `overrides` entry. A similar issue is possible here.

Additionally, the fallback package `@pasqal-io/starlight-client-mermaid` is also unpinned.

**Recommended Fix:**
1. Pin the primary package: `npm install astro-mermaid@1.3.1` (current latest at time of story creation).
2. Add to Dev Notes: "After install, run `npm ls zod` to verify no dual-instance conflict exists. If zod v4 appears in the tree, add it to the `overrides` block in `package.json` as done in 19.1."
3. Pin the fallback: `@pasqal-io/starlight-client-mermaid` — check current version on npm before documenting.
