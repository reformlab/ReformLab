### 1. Potential for YearState Metadata Mutation (Architectural Breach)

**Impact:** If implemented by direct modification, this would violate the immutability of `YearState` (a frozen dataclass), leading to non-deterministic behavior, difficult-to-trace bugs, and a breach of a core project architectural principle.
**Source:** AC-8, Task 4.4, project-context.md#critical-dont-miss-rules -> All domain types are frozen

**Problem:**
AC-8 and Task 4.4 instruct to 'extend existing `DISCRETE_CHOICE_METADATA_KEY` dict'. While 'Dev Notes' emphasize `YearState` immutability via `replace()`, the direct phrasing of 'extending' could lead a developer to mutate the dictionary object retrieved from `YearState.data`, violating the frozen dataclass principle and `replace()` update pattern.

**Recommended Fix:**
Clarify that extending the metadata dictionary must involve creating a *new* dictionary (e.g., using `dict` unpacking like `{**existing_metadata, 'new_key': new_value}`) and then updating the `YearState` with this new dictionary using `dataclasses.replace()`.

