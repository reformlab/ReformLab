### 1. AC-1 Contradicts Demo Scenario Definition

**Impact:** Developers cannot satisfy AC-1 and Task 1.2 simultaneously; test outcomes will be disputed.  
**Source:** AC-1, Task 1.2

**Problem:**  
AC-1 requires Stage 1–3 prefilled with a valid portfolio reference. Task 1.2 explicitly sets `portfolioName: null` and says no portfolio required for run.

**Recommended Fix:**  
Choose one approach and align all sections:
- Option A: Demo has a valid portfolio reference and corresponding seeded portfolio data.
- Option B: Template-only demo is allowed; rewrite AC-1 to remove portfolio-reference requirement.

### 2. Restore/Persist Ordering Can Destroy Returning State

**Impact:** Returning users can lose saved scenario/stage at login, violating AC-2 and AC-5.  
**Source:** Tasks 3.2–3.5

**Problem:**  
Persistence effects (`saveScenario`, `saveStage`) may run with default/null values before restore logic completes after auth transition.

**Recommended Fix:**  
Add explicit hydration phase:
- `isHydrating=true` during restore.
- Block persistence effects until hydrate completes.
- Persist only after first successful restore/init decision.
