## ⚡ Enhancement Opportunities (Should Add)

Additional guidance that would significantly help the developer avoid mistakes.

### 1. Robust fallback for `getHelpEntry`

**Benefit:** Provides more accurate contextual help or indicates absence of help gracefully for future `viewModes`, improving user experience.
**Source:** Step 4

**Current Gap:**
The `getHelpEntry` function currently falls back to `configuration:population` if a given `viewMode` (that is not `configuration`) or `activeStep` combination is not found in `HELP_CONTENT`. This means that if a new `viewMode` is introduced in the future and its help content is not yet defined, the user will see help for "Select Population," which is semantically incorrect and potentially confusing.

**Suggested Addition:**
Introduce a generic `default-help` entry in `HELP_CONTENT` (e.g., "General Help" with tips on navigating or finding documentation). Modify the `getHelpEntry` function to use this `default-help` entry as a final fallback when a specific entry for an unknown `viewMode` or `activeStep` is not found.

### 1. Explicit `HELP_CONTENT` object in Dev Notes

**Issue:** Context Parsing / Information Extraction
**Token Impact:** Potentially increases tokens slightly, but improves accuracy and reduces computation for the LLM.

**Current:**
```
Define the following interface and record. All text content is provided below — copy verbatim:

interface HelpEntry { ... }
function getHelpEntry { ... }

| Key | Title | Summary | Tips | Concepts |
|-----|-------|---------|------|----------|
... (table of entries) ...

Full tips and concepts per entry:
... (detailed bulleted lists) ...
```

**Optimized:**
```typescript
// frontend/src/components/help/help-content.ts
export interface HelpEntry {
  title: string;
  summary: string;
  tips: string[];
  concepts?: Array<{ term: string; definition: string }>;
}

export const HELP_CONTENT: Record<string, HelpEntry> = {
  "data-fusion": {
    title: "Population Builder",
    summary: "Create a synthetic population by selecting and merging data from multiple statistical sources.",
    tips: [
      "Select at least two data sources from the available providers to begin",
      "Overlapping variables (shared across sources) enable statistical matching for higher-quality fusion",
      "The merge method determines how records are combined — conditional sampling preserves correlations best",
      "Preview the generated population to verify demographic distributions before proceeding"
    ],
    concepts: [
      { term: "Data Fusion", definition: "Combining records from multiple data sources into a unified population dataset using statistical matching." },
      { term: "Overlapping Variables", definition: "Variables present in multiple sources that serve as matching keys for statistical fusion." },
      { term: "Conditional Sampling", definition: "A merge method that preserves correlations between variables by sampling conditionally on shared keys." }
    ]
  },
  // ... all other entries in full TypeScript object format
};

export function getHelpEntry(viewMode: string, activeStep?: string): HelpEntry {
  const key = viewMode === "configuration" && activeStep
    ? `configuration:${activeStep}`
    : viewMode;
  return HELP_CONTENT[key] ?? HELP_CONTENT["configuration:population"]!; // (with suggested enhancement for better fallback)
}
```

**Rationale:** Providing the `HELP_CONTENT` directly as a TypeScript object in the Dev Notes would eliminate the need for the LLM to parse and reconstruct this complex data structure from a table and separate bulleted lists. This reduces potential parsing errors and cognitive load, leading to more accurate and efficient implementation.
