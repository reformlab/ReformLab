### 1. Add state management pattern for population metadata fetching

**Benefit:** Prevents async state management bugs and provides clear implementation pattern
**Source:** AC-3, analysis of existing codebase

**Current Gap:**
The story doesn't explain how to add population metadata fetching to `PoliciesStageScreen`. Looking at existing patterns (categories state from Story 25.1), there's a model to follow, but it's not referenced.

**Suggested Addition:**
Add to Dev Notes:

```typescript
// Population metadata state pattern (follows Story 25.1 categories pattern)
const [populationSummaries, setPopulationSummaries] = useState<PopulationSummary[]>([]);
const [populationMetadataLoading, setPopulationMetadataLoading] = useState(true);

useEffect(() => {
  const fetchPopulationMetadata = async () => {
    if (!activeScenario.populationIds || activeScenario.populationIds.length === 0) {
      setPopulationSummaries([]);
      setPopulationMetadataLoading(false);
      return;
    }

    try {
      const summaries = await Promise.all(
        activeScenario.populationIds.map(id => getPopulationSummary(id))
      );
      setPopulationSummaries(summaries);
    } catch (error) {
      console.error("Failed to fetch population metadata:", error);
      // Graceful degradation: no warnings if metadata unavailable
      setPopulationSummaries([]);
    } finally {
      setPopulationMetadataLoading(false);
    }
  };

  fetchPopulationMetadata();
}, [activeScenario.populationIds]);
```
