### 1. Type Format Mismatch Bug - Will Cause Display Failures

**Impact:** Existing carbon_tax templates will not display correct labels or colors in PortfolioTemplateBrowser
**Source:** frontend/src/components/simulation/PortfolioTemplateBrowser.tsx lines 18-30 vs backend src/reformlab/server/routes/templates.py line 98

**Problem:**
- Backend API returns `type: "carbon_tax"` (underscore format) from `policy_type.value`
- PortfolioTemplateBrowser TYPE_LABELS has key `"carbon-tax"` (hyphen format)
- Lookup fails: `TYPE_LABELS["carbon_tax"]` → undefined
- Result: Falls back to displaying raw type name "carbon_tax" with default gray color

**Code Evidence:**
```typescript
// PortfolioTemplateBrowser.tsx line 18-23
const TYPE_LABELS: Record<string, string> = {
  "carbon-tax": "Carbon Tax",  // ❌ Backend returns "carbon_tax"
  "subsidy": "Subsidy",
  "rebate": "Rebate",
  "feebate": "Feebate",
  // Missing: "carbon_tax", "vehicle_malus", "energy_poverty_aid"
};
```

**Recommended Fix:**
Adopt PortfolioCompositionPanel's dual-format pattern:
```typescript
const TYPE_LABELS: Record<string, string> = {
  "carbon-tax": "Carbon Tax",
  "carbon_tax": "Carbon Tax",  // Add underscore format
  "subsidy": "Subsidy",
  "rebate": "Rebate",
  "feebate": "Feebate",
  "vehicle_malus": "Vehicle Malus",  // Use underscore format
  "energy_poverty_aid": "Energy Poverty Aid",  // Use underscore format
};
```
