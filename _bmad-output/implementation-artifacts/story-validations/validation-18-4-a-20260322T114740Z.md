## 🚨 Critical Issues (Must Fix)

These are essential requirements, security concerns, or blocking issues that could cause implementation disasters.

### 1. Conflicting Metadata Source-of-Truth

**Impact:** High risk of shipping wrong run metadata (policy label/year range) in results header.  
**Source:** Story AC-1 and Dev Notes props contract.

**Problem:**  
AC-1 requires policy label derivation based on result detail (`portfolio_name > template_name > fallback`), but the proposed props design pushes a scenario-derived `reformLabel` from `App.tsx`. These can diverge from actual run metadata.

**Recommended Fix:**  
Define a single metadata contract in AC and tasks:
- `policyLabel` derived from result detail when available.
- explicit fallback chain when detail unavailable.
- year range source fields and fallback.
Update test cases to validate this precedence.

### 2. Stale Detail Cache Across Runs

**Impact:** High trust failure: user can view detail from previous run after starting/selecting another run.  
**Source:** Task 4.2 and loadDetail snippet.

**Problem:**  
Story instructs caching detail to avoid refetch but does not specify invalidation on `runResult.run_id` change.

**Recommended Fix:**  
Require cache keyed by `run_id` and reset behavior:
- clear detail on run change,
- fetch once per run id,
- show loading/error state per run.
Add explicit tests for run switch and stale-data prevention.
