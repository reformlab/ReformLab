## Issues Dismissed

- **Claimed Issue**: Module-level mutable `_DEFAULT_OUTPUT_MAPPING` violates frozen-first convention | **Raised by**: Validator A | **Dismissal Reason**: The project rule "frozen dataclasses are the default" applies to domain types (dataclasses), not module-level constants. A `dict` constant at module scope is standard Python practice. The constant is intended as read-only and documented accordingly. No real risk — no code mutates it. Using `MappingProxyType` would add complexity for zero practical benefit.

- **Claimed Issue**: Test file `tests/computation/test_normalization_regression.py` in wrong directory | **Raised by**: Validator A | **Dismissal Reason**: The normalizer module is in `computation/`, and the regression tests test computation normalization output against indicator expectations. `tests/computation/` is the correct mirror location. The tests don't test `orchestrator/` or `indicators/` directly — they verify the normalizer produces indicator-compatible output.

- **Claimed Issue**: Split story into contract-first + wiring stories | **Raised by**: Validator B | **Dismissal Reason**: The story is 8SP. Splitting would create two 4SP stories with duplicated context and overhead. The fixes applied (removing NormalizedOutputSchema, clarifying metadata flow) resolve the ambiguity that made the story feel too large.

- **Claimed Issue**: Add export regression tests | **Raised by**: Validator B | **Dismissal Reason**: Export flow is mentioned in the story statement as a consumer, but the task list already covers the two primary consumers (indicators, comparison). Export compatibility follows from correct panel schema normalization. Adding export-specific tests would expand scope without testing a different code path.

- **Claimed Issue**: Legacy result compatibility tests | **Raised by**: Validator B | **Dismissal Reason**: Story 23.3 adds normalization for live mode only. Existing stored results (all replay/mock) are unaffected — the normalizer is only applied when `runtime_mode == "live"`. No migration needed.
