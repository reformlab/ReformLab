## 🚨 Critical Issues (Must Fix)

### 1. AC Contradiction on UI Parity vs Normalization

**Impact:** Developers cannot satisfy both constraints objectively; review outcomes become arbitrary.  
**Source:** AC-4 + ErrorAlert Dev Notes

**Problem:** Story requires identical visual output while instructing normalization toward a canonical pattern that changes icon sizing/spacing/labels in some screens.

**Recommended Fix:** Split into explicit rule set:  
- Rule A: Interaction parity must remain identical.  
- Rule B: Visual parity requirements per screen (strict/allowed deltas).  
- Rule C: If normalization is desired, update AC-4 to permit specified visual deltas.

### 2. Unsafe Dead-Code Deletion Criteria

**Impact:** High regression risk from deleting live-but-indirectly-used files.  
**Source:** AC-5 + Task 4.1/4.2

**Problem:** “Confirmed unused via grep” is insufficient proof in modern TS apps (barrels, route maps, dynamic imports, test-only references).

**Recommended Fix:** Require deletion gate checklist:  
`rg` import search + exports/barrel check + route/config check + test references + successful typecheck/lint/test after deletion.
