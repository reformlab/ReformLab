## 🚨 Critical Issues (Must Fix)

### 1. Missing Pipeline Contract for Per-Domain Recording

**Impact:** Multi-domain runs can lose earlier domain records.  
**Source:** AC-2, Tasks 1/6

**Problem:** The story does not require concrete pipeline changes to insert `DecisionRecordStep` after each domain update.

**Recommended Fix:** Add explicit AC and test asserting pipeline contains one decision-record step per domain and that both records are present in-year in deterministic order.

### 2. Incorrect Manifest Semantics for Year-Varying Seed

**Impact:** Manifest can be misleading for reproducibility audits.  
**Source:** AC-5

**Problem:** Capturing from first year assumes invariance for fields that may vary yearly (`choice_seed`).

**Recommended Fix:** Capture static params once (`beta_cost`) and capture per-year seed separately, or remove `choice_seed` from first-year-only extraction.

### 3. Missing Data Integrity Invariants in Panel Injection

**Impact:** Runtime failure or corrupted decision columns.  
**Source:** AC-3, AC-6

**Problem:** No mandatory checks for row alignment or duplicate `domain_name` collision behavior.

**Recommended Fix:** Add required validations and explicit failure semantics with year/domain context.

### 4. No Scale Guardrail for Decision Column Materialization

**Impact:** Severe performance/memory regressions on large populations.  
**Source:** AC-3, AC-4

**Problem:** ACs require rich list columns but do not enforce performant construction.

**Recommended Fix:** Add performance AC (dataset size + max runtime/memory envelope) and require Arrow-native/vectorized path.
